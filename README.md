# LeetCode 刷题记录

## 介绍
这是我在 LeetCode 上刷题的记录，包括解法、反思与心得。我会按照题目的难度分类，将每一道题的解法、优化思路、反思和时间空间复杂度分析等记录下来，帮助自己在刷题过程中总结和提升，也希望能与其他刷题的同学们分享。

## 刷题计划
- 每周至少刷 5 道题目，涵盖不同类型和难度。
- 针对每道题目，进行详细的解法分析，包括时间复杂度、空间复杂度、算法优化等。
- 在刷题过程中，记录自己的反思和总结，尤其是对一些容易忽视的细节或新的算法思想的理解。
- 完成 LeetCode 上的经典题目（如排序、动态规划、二分查找、回溯等）。

## 项目结构
- **`solutions/`**: 存放具体的题解文件，按照题目的难度分类（easy, medium, hard）。每道题目一个独立的 Markdown 文件。
- **`docs/`**: 存放刷题过程中的总结文档，包括不同题型的总结、反思与心得，以及学习过程中使用的参考资源。
- **`LeetCode.md`**: 记录刷题进度和目标，跟踪当前进度，计划未来学习方向。

## 题目分类
- **Easy**: 简单题目，适合快速上手，增强对基本数据结构和算法的掌握。
- **Medium**: 中等难度题目，涉及较复杂的算法和技巧，提升解决问题的能力。
- **Hard**: 困难题目，挑战自己的思维和算法设计能力，处理更复杂的场景和约束。

## 刷题工具
- **LeetCode**: [LeetCode 官方网站](https://leetcode.com/)
- **GitHub**: 用于记录和分享题解及反思。

## 参与贡献
欢迎大家通过提交 Issue 或 Pull Request 来贡献你的想法或题解。如果你有更好的解法，或者发现了我解法中的问题，欢迎指出。

## 反思与心得
刷题不仅仅是为了求解一个个问题，更是一个自我提升的过程。在不断的刷题和总结中，我慢慢地掌握了很多算法技巧和思路，也意识到自己在算法设计和优化上的不足。每当我解决一个新的问题，都会进行反思，思考如何在下次遇到类似问题时能更加高效地解决。

希望这个项目能够为大家提供一些有价值的帮助，也希望能与更多的刷题伙伴一起交流学习。
